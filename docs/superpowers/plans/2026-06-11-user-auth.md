# User Auth & Multi-Tenant Isolation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Phase 1 — email login gate, camelCase `users` table, all portfolio/report data scoped by `userId`, Web login/register pages.

**Architecture:** FastAPI middleware validates JWT, sets `userId` in ContextVar; `database.py` filters all queries by `userId`; SQLite migrations assign legacy data to user `id=1`; Next.js stores `accessToken` and redirects unauthenticated users.

**Tech Stack:** FastAPI, PyJWT, bcrypt, SQLite (camelCase user columns), Next.js App Router

**Spec:** `docs/superpowers/specs/2026-06-11-user-auth-design.md`

---

## File Map

| File | Responsibility |
|------|----------------|
| `app/request_context.py` | ContextVar `userId` per request/thread |
| `app/auth/models.py` | Pydantic: Register, Login, UserPublic, TokenResponse |
| `app/auth/passwords.py` | bcrypt hash/verify |
| `app/auth/jwt.py` | encode/decode access token |
| `app/auth/service.py` | register, login, get_user_by_id |
| `app/auth/middleware.py` | HTTP middleware: public paths + JWT |
| `app/db_migrations.py` | Schema v2: users, userId on business tables |
| `app/database.py` | User CRUD + all queries scoped by `_uid()` |
| `app/main.py` | `/api/auth/*` routes |
| `app/services/job_store.py` | Store `userId` on jobs, set context in worker |
| `tests/conftest.py` | JWT secret, `auth_client` fixture |
| `tests/test_auth.py` | Register, login, isolation tests |
| `web/src/lib/auth.ts` | Token storage |
| `web/src/lib/api.ts` | `apiFetch` with Authorization header |
| `web/src/components/AuthProvider.tsx` | Auth state + redirect |
| `web/src/app/login/page.tsx` | Login form |
| `web/src/app/register/page.tsx` | Register form |
