# 关联板块计算式解析设计

日期：2026-06-30

## 背景

当前 OCR 录入持仓后，部分新基金的“关联板块”会长时间显示为空，且命中精度不够。已排查到两个主要原因：

- OCR 确认链路为了速度走快路径，默认不抓取基金基准和持仓穿透。
- 当前名称推断主要依赖少量主题别名，覆盖不足，且在关键快路径里不是稳定启用的核心能力。

竞品截图显示，养基宝等工具并不只等待行情板块匹配成功才展示结果，而是会先展示一个可解释的关联标签。例如：

- 华夏中证电网设备主题 ETF 联接 C：中证电网设备/电网设备
- 天弘科创芯片设计 ETF 联接 C：科创芯片设计
- 天弘全球高端制造混合(QDII)C：全球高端制造
- 广发全球精选股票(QDII)人民币 C：全球精选股票
- 富国全球科技互联网股票(QDII)C：海外基金

本次目标是用“计算式解析 + 多信号融合”替代“名称白名单主导推断”，提升 OCR 录入后的即时可用性和长期准确性。

## 目标

1. OCR 确认后立即给新基金生成尽可能准确的关联板块标签。
2. 不再把主题白名单作为主要推断机制；白名单仅用于同义词、简称、行业名归一。
3. 兼容竞品口径：QDII/海外类基金可以按“海外基金”或更明确的全球主题归类。
4. 保持快路径速度：OCR 确认不阻塞在慢网络请求上。
5. 保留现有高置信来源优先级，不覆盖用户手动选择或高可信 OCR 明细。

## 非目标

- 不在本次实现里做完整基金持仓穿透算法重构。
- 不引入大模型在线调用作为 OCR 确认链路依赖。
- 不要求所有推断结果都能立刻拿到板块行情；没有行情映射时也先展示标签。

## 方案

新增一个“基金板块语义解析器”，输入基金名称、基金代码和可选来源信息，输出候选结果：

```text
sector_name: 用于展示和持仓记录的关联板块
source: semantic_name / benchmark_index / holdings_infer / manual / ocr_detail 等
confidence: 0-100
reason: 简短可测试的命中原因
quote_key: 可选，用于匹配行情板块
```

解析器不直接依赖固定主题白名单，而是按以下顺序计算候选：

1. 清洗基金名中的产品噪音：基金公司、ETF、联接、发起式、指数、混合、股票、QDII、人民币、份额字母等。
2. 从剩余名称中提取主题短语，优先保留“科创芯片设计”“全球高端制造”“全球精选股票”等连续语义片段。
3. 使用现有主题注册表、行情板块表和规范化工具做相似度匹配。
4. 匹配到行情板块时返回可报价板块；无法匹配行情但短语可信时，仍返回展示标签。
5. QDII/海外类基金使用单独规则：
   - 海外科技类泛主题优先归为“海外基金”。
   - 名称中存在明确资产主题时保留主题，例如“全球高端制造”“全球精选股票”。

## 数据流

OCR 确认链路：

1. 用户上传截图并确认持仓。
2. 后端解析出基金名称、金额、可选基金代码。
3. 保存持仓前调用板块解析服务。
4. 快路径只使用本地语义解析、已有缓存和已有全局映射，不等待慢网络请求。
5. 若解析出 sector_name，立即写入持仓记录。
6. 行情刷新若找不到 quote_key，仍保留 sector_name 展示，行情列可继续显示空值或等待后台补齐。

后台增强链路：

1. 常规刷新或非快路径可继续抓取基准指数、基金档案、持仓穿透。
2. 高置信基准/持仓结果可升级低置信语义结果。
3. 手动设置和高可信 OCR 明细始终优先，不被自动推断覆盖。

## 优先级

自动解析结果进入现有 `resolve_primary_sector` 优先级体系：

1. manual / ocr_detail：最高优先级。
2. benchmark_index：强信号，优先于名称语义。
3. semantic_name：本次新增的即时语义结果。
4. holdings_infer：后台穿透结果，可在置信度更高时升级。
5. legacy name_infer：保留为兼容兜底，但不作为主路径。

## 错误处理

- 无法提取可信主题时返回空，不强行给低质量标签。
- 多个候选分数接近时，优先更长、更具体、且存在注册表或行情表支持的主题。
- 如果只有泛词，例如“精选”“灵活配置”“成长”，不单独作为关联板块。
- 网络异常不影响 OCR 确认保存；后台刷新失败只影响行情补齐。

## 测试策略

先写失败测试，再实现：

1. 语义解析单元测试：覆盖基金名清洗、主题提取、QDII 归类、噪音过滤。
2. 服务集成测试：`fetch_benchmark=False` 时也能为 OCR 新持仓即时写入 `sector_name`。
3. 回归测试：手动板块和高可信 OCR 明细不会被语义推断覆盖。
4. 性能/链路测试：OCR 快路径不触发慢基准抓取，仍能完成新基金关联板块填充。

## 验收标准

- 用户给出的 6 只基金在 OCR 确认后无需长时间等待即可显示预期关联板块。
- 没有可报价行情的语义标签也能正常展示，不再直接显示为空。
- 现有基金基准解析、持仓刷新和用户手动设置行为不回退。
- 目标测试通过；若存在 unrelated 旧失败，需要在验收说明中明确列出。
