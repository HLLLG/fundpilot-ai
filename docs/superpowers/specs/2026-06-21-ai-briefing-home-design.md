# 好基灵 AI 简报首页 — 设计方案（方案 B）

**日期：** 2026-06-21  
**状态：** 已确认，实施中  
**前置：** `2026-06-20-haojiling-toc-redesign-design.md` 第 0+1 批已完成

---

## 1. 目标

将登录后默认首屏从「持仓表格」升级为 **AI 简报首页**（参考蚂蚁财富蚂小财简报模式），让用户第一眼看到：

1. 今日收益与总资产（金融 KPI）
2. AI 一句话摘要（来自最新日报 `report.summary` 或操作建议）
3. 板块脉搏（持仓关联板块中最值得关注的 1 条）
4. 持仓快览（Top 3，卡片化）
5. 明确行动：查看全部持仓 / 生成完整日报 / 上传截图

**不改变**后端 API、数据流、涨跌色语义、静态导出能力。

---

## 2. 信息架构变更

| 原 Tab | 新 Tab | 说明 |
|--------|--------|------|
| 持有 (`today`) | **简报** (`today`) | 新 `TodayBriefing` 组件，默认首屏 |
| — | **持仓** (`holdings`) | 原 `YangjibaoHoldingsBoard` 完整看板 |
| 盈亏分析 | 分析 (`dashboard`) | 文案精简 |
| 市场 | 市场 (`market`) | 不变 |
| 推荐基金 | 发现 (`discovery`) | 文案精简；移动端收入「更多」 |
| 生成日报 | 日报 (`report`) | 移动端收入「更多」 |

**移动端导航（底部固定 5 格）：** 简报 · 持仓 · 分析 · 市场 · 更多（发现 / 日报 / 历史）

**桌面端：** 顶部分段 Tab 展示全部 6 项（可横向滚动）。

## 3. 简报页模块

### 3.1 英雄区（Hero）

- 问候语：`早上好 / 下午好 / 晚上好` + 交易日标签
- 主 KPI：当日收益（大数字，红涨绿跌，可切换收益率）
- 副 KPI：总资产（支持隐藏金额，复用 `loadAmountsHidden`）
- 刷新按钮：触发板块行情 refresh

### 3.2 AI 简报卡

**有今日日报时：**
- 标题「今日 AI 简报」+ 风险等级 pill + 建议动作 pill
- 正文：`report.summary` 首段（超 120 字截断 + …）
- 若有 `fund_recommendations[0]`：附一行「重点关注：{基金名} · {action}」
- CTA：「阅读完整日报」→ 跳转 `report` Tab

**无今日日报时：**
- 占位文案 + 说明价值
- CTA：「生成今日日报」→ 跳转 `report` Tab 并 scroll 到分析控件

### 3.3 板块脉搏

从 displayable holdings 中取 `sector_return_percent` 绝对值最大的板块，展示：
- 板块名 + 涨跌幅 + 「连涨 N 天」若 meta 有 `consecutive_up_days`

无板块数据时隐藏此卡。

### 3.4 持仓快览

- Top 3（按当日收益额降序）
- 卡片：基金名、板块 chip、当日收益、持仓金额
- 底部：「查看全部 N 只持仓」→ `holdings` Tab

### 3.5 空状态（无持仓）

复用现有截图引导逻辑，嵌入简报页居中展示。

## 4. 视觉规范

延续「静谧蓝海」token，新增：

- `.briefing-hero` — 渐变英雄底
- `.briefing-ai-card` — AI 简报主卡（品牌蓝左边条 + 柔光）
- `.briefing-holding-chip` — 快览持仓行
- `.dashboard-bottom-nav` — 移动端底栏（safe-area 适配）

入场动画：`.reveal` 分级延迟（与落地页一致）。

---

## 5. 验收标准

- 登录后默认见简报页，非持仓表格
- 有日报时展示 AI 摘要；无日报时展示生成引导
- 移动端底栏可切换全部主功能；桌面顶栏 6 Tab 正常
- `lint` / `typecheck` / `test` / `build` 全绿
- 现有 e2e / vitest 不回归
