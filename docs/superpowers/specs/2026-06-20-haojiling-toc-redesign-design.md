## 1. 背景与目标

FundPilot AI 核心功能（OCR 识别持仓、板块涨跌估算、DeepSeek 投研日报、推荐基金）已完成。当前前端是一套干净但偏"功能工具"气质的界面（Next.js 16 / React 19 / Tailwind v4 / Plus Jakarta Sans）。

本次改造目标：**把产品从"功能工具"升级为面向普通基民（小白）的、好看、亲和、有安全感、能促成注册转化的 toC 订阅产品**，让更多人愿意使用，为后续商业化打基础。

### 决策记录（用户确认）

| 项 | 决策 |
|----|------|
| 商业定位 | toC 订阅产品，面向普通基民小白，强调"上手简单、好看、有安全感"，需要营销门面与引导 |
| 改造范围 | 全量改造，**分批排优先级**推进，每批做完用户本地实时预览验收 |
| 品牌气质 | 信任蓝 + 暖橙点缀（延续现有蓝色品牌，更通透友好） |
| 涨跌色 | **保持 A 股习惯：红涨 `#E11D48` / 绿跌 `#059669`（不变，数据正确性）** |
| 产品名 | 中文「好基灵」为主 + 英文「FundPilot」辅助 |
| Slogan | 好基灵，截个图就懂你的基金 |
| 副标语 | 自动识别持仓明细 · 实时追踪板块冷暖 · 每天一份听得懂的投研日报 |
| 落地页 CTA | 方案 A：免费注册优先（先跑通获客漏斗，暂不展示付费引导） |

## 2. 设计语言（统一规范，贯穿所有页面）

### 2.1 配色 token

在现有 `globals.css` 的 `:root` 变量基础上扩展（保持向后兼容，旧变量名保留并指向新值）：

| 角色 | 变量 | 值 | 用途 |
|------|------|----|------|
| 品牌主蓝 | `--brand` | `#2563EB` | 主按钮、链接、品牌、激活态 |
| 品牌深蓝 | `--brand-strong` | `#1D4ED8` | hover/按下、渐变深端 |
| 品牌浅底 | `--brand-soft` | `#EFF4FF` | 选中底、信息条、标签底 |
| 暖橙点缀 | `--accent` | `#FB8C3B` | 强调标签、引导高亮、次要 CTA（克制使用） |
| 暖橙浅底 | `--accent-soft` | `#FFF3E8` | 暖色提示底 |
| 涨（红） | `--profit-up` | `#E11D48` | 涨幅、正收益（**不变**） |
| 跌（绿） | `--profit-down` | `#059669` | 跌幅、负收益（**不变**） |
| 前景文字 | `--foreground` | `#0F172A` | 主文字 |
| 次级文字 | `--muted` | `#64748B` | 辅助文字 |
| 背景 | `--background` | `#F6F8FB` | 页面底（略暖中性） |
| 卡片 | `--panel` | `#FFFFFF` | 卡片/面板 |
| 描边 | `--line` | `rgba(15,23,42,0.07)` | 分隔线、卡片描边 |

> 兼容性：现有组件大量使用 Tailwind `blue-600` 等硬编码色。第 0 批先统一 token，但**不强制**一次性替换所有硬编码蓝；以"新建/重做的页面用 token，存量页面逐批迁移"为原则，避免大范围回归风险。

### 2.2 圆角 / 阴影 / 间距

- 圆角：卡片 `20px`（`--radius-card`），按钮胶囊化（`9999px` 或 `14px`），输入框 `12px`。
- 阴影：分层柔和阴影 token：`--shadow-sm`（卡片静态轻浮）、`--shadow-md`（hover 抬升）、`--shadow-lg`（落地页/弹窗）。
- 间距：增大留白与行高，降低信息密度（小白向）。区块间距以 8 的倍数为节奏。

### 2.3 字体

- 拉丁/数字：继续 `Plus Jakarta Sans`（`--font-sans`）。
- 中文：补齐字体栈 `"PingFang SC", "Microsoft YaHei", ...`（已部分存在，统一规范）。
- 字号层级 token：`display`（落地页大标题）、`h1/h2/h3`、`body`、`caption`，配套字重与字间距。

### 2.4 共用组件类（在 `globals.css` 增补）

- 按钮：`.btn-primary`（品牌蓝渐变 + 阴影）、`.btn-secondary`（白底描边）、`.btn-ghost`、`.btn-accent`（暖橙，次 CTA）；统一高度、圆角、hover/active/disabled 态。
- 卡片：升级 `.section-card`（更圆、分层阴影、hover 抬升可选）。
- 标签/徽章：`.badge`、`.badge-accent`、`.badge-up`/`.badge-down`。
- 区块标题：`.section-title` 升级（带品牌竖条/图标位）。
- 空状态：`.empty-state` 容器规范（插画/图标位 + 标题 + 说明 + 行动按钮）。
- 微交互：保留并扩展 `animate-fade-up`，新增 hover 抬升与 Tab 过渡。

### 2.5 合规

toC 金融门面必须含风险提示。落地页底部 + 注册页 + 日报相关位置统一文案：**"投资有风险，入市需谨慎。本工具提供的内容仅供参考，不构成任何投资建议。"**

## 3. 路由与登录态（落地页接入）

**现状：** `output: "export"`（纯静态导出，客户端路由）。`AuthProvider` 中未登录访问 `/` 会被 `router.replace("/login")` 踢走；`/login` `/register` 为 `PUBLIC_PATHS`。

**改造：**
- 将 `/` 纳入"公开可访问"：未登录访问 `/` 展示**品牌落地页**，已登录展示 `Dashboard`。
- 实现方式（静态导出安全）：
  - `AuthProvider`：把 `/` 加入允许公开访问的逻辑（未登录在 `/` 不再强制跳 `/login`）。
  - `app/page.tsx`（`Home`）：用 `useAuth()` 判断——`user` 存在 → 渲染 `<Dashboard />`；否则渲染 `<LandingPage />`。
  - `/login` `/register` 保持不变；已登录访问这两页仍重定向回 `/`。
- 受保护的具体功能页（如未来子路由）仍走原有"无 token → 跳登录"逻辑。

> 实现时确认：`Home` 改为客户端分支渲染后，静态导出仍只产出 `/`、`/login`、`/register`、`/settings` 等既有页面，不新增需要 SSR 的动态路由。

## 5. 约束与非目标

- **不改动后端 API、数据流、业务逻辑**；纯前端表现层 + 文案 + 新增落地页。
- **不破坏静态导出**（`output: "export"`）。
- **不改变涨跌色语义**与持有列展示口径（遵循 `holding-metrics-contract`）。
- **不引入重型 UI 库**（继续 Tailwind + lucide-react + 现有依赖），避免膨胀与回归。
- 不做付费/订阅页（CTA 选 A，仅获客；付费引导留待后续）。
- 保持中文文案为主，语气亲和、说人话。
