# 量化试运行（一个月版）

## 目标与边界

量化试运行用于在严格 PIT v3 尚未积累足够历史快照时，提前验证当前 `factor_ic.v2` 是否有产品价值。它不是正式量化证据，只能对已经通过板块、资金、持仓、交易与风险守卫的首批买入金额做最多 `±10%` 的修正。

它不得：

- 把“建议关注”或“等待回调”升级为“分批买入”；
- 绕过现金、总预算、板块集中度、申购限额或费用门禁；
- 替代严格 PIT v3 的执行资格；
- 宣称回测消除了幸存者偏差或证明未来收益。

## 运行模式

| 模式 | 行为 | 配置 |
| --- | --- | --- |
| 关闭 | 不生成试运行契约 | `FUND_AI_FACTOR_PREVIEW_MODE=off` |
| 影子（默认） | 记录基础金额与反事实金额，不改变本次建议 | `FUND_AI_FACTOR_PREVIEW_MODE=shadow` |
| 灰度 | 仅对可执行首批金额应用受限修正 | `FUND_AI_FACTOR_PREVIEW_MODE=enforced` |

最大幅度由 `FUND_AI_FACTOR_PREVIEW_MAX_ADJUSTMENT_PERCENT` 配置，服务端会强制限制在 `0～10`。

## 可审计输入与准入

每条试运行结果必须绑定 `snapshot_id`、`run_date`、模型版本、样本口径、基金代码和可复算因子分位。缺一项即失败关闭。

当前准入条件：

- 模型为 `factor_ic.v2`，样本口径为 `current_survivors`；
- IC 快照可用、未过期，且有快照编号与数据日期；
- 基金进入本次线上因子计算集合；
- 同类有效样本不少于 20，只使用至少 2 个因子且完整度不低于 50%；
- 目标基金净值特征为 `fresh`，收益序列覆盖不低于 90%；
- 至少一个因子的回测置信达到“中”，低置信或未回测因子不参与试运行分数。

试运行分数是合格因子分位的等权平均，并在当前候选池的同板块范围内给出展示性排名。排名样本数会一并展示，避免把 `1/1` 误读为有统计意义的第一名。

## 固定金额映射

映射在观察期开始前冻结，避免根据结果反复调参：

| 试运行分数 | 建议修正 |
| --- | ---: |
| `≥ 85` | `+10%` |
| `70～84.99` | `+5%` |
| `30.01～69.99` | `0%` |
| `15.01～30` | `-5%` |
| `≤ 15` | `-10%` |

实际修正还会被配置上限、现金、预算、板块集中度和单日申购限额再次截断。若后续交易费用或任何证据门禁失败，最终动作降级，已计算的试运行修正同步回退为“未应用”。

## 一个月启用节奏

1. 第 1～15 个交易日保持 `shadow`，日报/荐基正常记录基础金额、反事实金额、快照和后续结果。
2. 第 16～20 个交易日完成固定口径复核，不因短期盈亏临时更改阈值。
3. 满 20 个交易日后，满足下列条件可人工切换为 `enforced`：
   - 快照按期生成率不低于 90%；
   - 至少 30 条合格候选观察，且包含不止一个板块；
   - 动作升级、突破硬上限、使用过期快照三类违规均为 0；
   - 影子金额和最终金额可由报告契约逐条复算；
   - 已成熟的 5 日/10 日结果未显示明显反向，且扣除已知交易成本后没有系统性恶化。
4. 任一安全条件不满足，继续 `shadow`；这不阻止产品展示试运行分数，但禁止金额生效。

一个月后的切换只是“小额灰度可用”，不是“正式 v3 完成”。严格 PIT v3 仍按真实历史快照、锚点覆盖、样本外稳定性、多重检验和扣费后经济显著性独立验收。

## 产品文案

前端统一显示“量化试运行”，并明确：

- `影子`：展示反事实金额并注明本次未生效；
- `已应用`：展示首批金额从基础值到最终值的变化；
- 始终显示“当前存续样本 / 非正式 PIT v3 / 不改变动作”。

不展示不合格试运行的长篇技术原因，具体失败原因保留在结构化契约中供诊断和审计。
