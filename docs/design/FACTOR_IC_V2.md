# Factor IC v2：分类研究池与基金级证据

> 本文保留 2026-07-12 的 v2 生产基线。2026-07-13 起，发布器同时支持并优先尝试 schema v3；完整的成员 PIT、统计/经济双门槛、类型因子和线上校准契约见 [量化证据 V3](QUANT_EVIDENCE_V3.md)。v3 未成熟或不合格时自动保留/回退 v2，不能降低门槛伪装升级。

## 目标

Factor IC v2 用于回答“某只基金的某个因子，在同类基金、对应持有周期下是否具有稳定的样本外排序能力”。它不是交易信号，不单独触发加仓或减仓，只调整日报与荐基的量化证据置信度。

## v1 问题

- 仅从近一年排行榜前 500 个份额抽 300 个，约占源目录 1.5%，且属于偏强幸存者样本。
- 在线排行榜没有最大回撤和规模字段，所谓综合因子多数退化为动量；离线 IC 与线上评分特征不一致。
- 股票、混合、债券、指数、QDII、FOF 混在同一横截面。
- 单位净值比值可能把分红导致的净值下降误判成负收益。
- 只有未来 20 日单周期和普通 t 统计，没有时间相关修正、留出稳定性或基金级适用性。

## 数据流

1. GitHub Runner 分页读取东财开放式基金全目录，保留 `gp/hh/zq/zs/qdii/fof` 分类。
2. 用“类别 + 规范化基金名”保守合并 A/C/E 等份额，A 份额优先作为长历史代表；不确定时不跨类别合并。
3. 各类别先保底分配样本，再按可用组合数分配剩余名额；类别内按近一年业绩排序等距抽样，覆盖赢家到输家。
4. 对 1,500 个代表组合拉取 1,500 个净值观测；优先按 `日增长率` 重建总收益指数，缺失时才回落单位净值比值。
5. 每个因子锚点必须有完整 250 个观测；当前/未来净值超过类别允许的陈旧天数则该期退出。
6. 在六个同类组分别计算未来 5/20/60 日 Rank IC；再平衡步长 10 个净值日。
7. IC 统计输出均值、ICIR、HAC/Newey-West 标准误、95% 区间、末 1/3 留出均值/正向比例和方向稳定性。
8. 同一份总收益特征生成各类别线上评分分布；日报/荐基目标基金用自身净值计算相同因子，再只与自己的同类组比较。
9. v2 快照通过独立 Token 与 SSH 隧道发布到 Lighthouse MySQL；v1 快照继续可读，直到 v2 首次发布成功。

## 生产固定口径

```text
schema_version       = 2
source pool cap      = 25,000（实际以源目录为准）
sampled portfolios   = 1,500
nav observations     = 1,500
factor lookback      = 250
rebalance step       = 10
forward horizons     = 5, 20, 60
primary horizon      = 20
max workers          = 16
```

发布硬门槛：

- 源目录份额数不少于 5,000；
- 去重组合数不少于有效研究池；
- 有效总收益序列不少于 1,200；
- 总收益优先口径覆盖率不少于 80%；
- 至少四个基金类别同时具备分类 IC 与同类评分分布；
- 顶层兼容因子完整、有效时期不少于 12；分类因子用于 LLM 前须至少 24 个时期并有末段留出结果。

任一硬门槛失败时，本次任务失败且保留上一份有效快照。

## LLM 与 Guard 契约

每只基金收到：

- `peer_group` / `peer_group_label`；
- `peer_count`；
- `feature_count` / `feature_completeness`；
- `applicable`；
- 基金级 `factor_reliability`；
- 同类因子百分位与综合分。

约束：

- `applicable=false`、未知类别、历史不足或同类样本不足时 IC 不参与。
- 负的整体或留出 IC 标记为“反向/均值回归”，不得把高百分位解释成正面证据。
- v2 不再向 LLM 提供可误用的全局因子可靠性；可靠性作用域固定为 `per_fund_peer_group`。
- 当前基金池仍是 `current_survivors`，在积累足够 point-in-time 历史以前，正向稳定 IC 最高只给“中”。
- 规模因子没有历史规模序列，保持“未回测”。

## 仍然保留的诚实边界

- 免费源无法恢复已经消失基金的完整历史目录，因此当前仍有幸存者偏差；每周快照会从 v2 上线日起积累 point-in-time 基金池。
- 基金类别来自当前东财大类，不等同于持仓穿透后的精细风格；未知类别直接退出，不猜测。
- 分红总收益依赖源站日增长率质量；覆盖率和回落数量进入诊断。
- 历史规模、费用、经理任期和持仓风格漂移尚未进入 IC；在获得稳定 point-in-time 数据前不伪造这些因子。

## 2026-07-13 v3 增量

- 数据库 schema v11 新增不可变 `factor_ic_universe_snapshots / members`，工作日晚间捕获 1,500 个成员研究池；周任务读取不超过 180 个快照。
- 每个类型×因子×5/20/60 日周期使用 5 折 expanding walk-forward、对应标签 embargo、HAC 区间和 Benjamini-Hochberg FDR；统计期至少 30。
- 只有再通过 36 期经济门槛、0.5% 成本、分位单调性、尾部与样本外价差后才可 qualified。
- 当前真实边界是 `membership_only`、`nav_revision_pit=false`：普通基金因子 NAV 滞后 1 个交易日，QDII 滞后 2 日，未来收益从下一交易日首个可执行 NAV 起算，可靠性最高为“中”。状态徽标显示“成员PIT”，不会显示成“完整PIT”。
- 线上目标基金额外校验自身 NAV 新鲜度与收益覆盖；模型时点与目标特征时点分别冻结，陈旧/未来时点直接退出。
- v3 的 20 日经济门槛理论最短约 372 个交易日（17.5 个月），60 日约 412 个交易日（19.5 个月）；这是最短样本量，不保证届时通过显著性和净成本门槛。
